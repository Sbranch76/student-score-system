## 一、项目概述
### 1. 项目背景
传统学生成绩管理多采用 Excel 表格或人工纸质记录方式，普遍存在查询效率低、数据易出错、跨角色共享困难、统计分析繁琐等问题。为解决上述痛点，本项目拟开发一套轻量化的在线学生成绩管理系统，实现学生成绩的规范化、数字化管理，降低人工操作成本，提升成绩管理的整体效率。

### 2. 项目目标
1. 实现**多角色登录验证**（管理员、教师、学生），完成基础身份鉴权
2. 完成学生基础信息的增删改查（CRUD）核心操作
3. 实现成绩信息的增删改查（CRUD）核心操作
4. 支持按学号、课程名称、学生姓名、班级进行多条件组合成绩查询
5. 支持**学生登录查询本人成绩**，完善系统角色闭环
6. 保障系统核心功能可稳定运行、可基础演示
7. 搭建规范的项目开发目录结构，完成 Git 仓库基础配置

### 3. 开发环境与条件
- 后端：Spring Boot 3.x、Spring Web、Spring Data JPA
- 前端：HTML5、CSS3、JavaScript、Bootstrap、Vue
- 数据库：MySQL 8.0
- 工具：IntelliJ IDEA、Git、GitHub、Postman
- 运行环境：JDK 17、Maven 3.8+

## 二、需求分析
### 1. 目标用户与使用场景
- **核心目标用户**：系统管理员、课程教师、学生
- 次要目标用户：辅导员

- 学生：登录系统、查询本人各课程成绩、查看个人信息
- 教师：录入学生课程成绩、修改错误成绩、查询所教课程成绩、成绩统计
- 管理员：维护学生/教师信息、维护课程信息、核验成绩数据、批量查询成绩

### 2. 核心功能清单
1. **多角色用户登录**（管理员、教师、学生）
2. 学生信息增删改查
3. 教师信息增删改查
4. 课程信息增删改查
5. 成绩信息增删改查
6. 多条件成绩查询功能
7. 学生个人成绩查询功能
8. 基础权限控制

### 3. 非功能需求
- 性能：接口响应时间≤1秒
- 安全：仅登录用户可操作，不同角色权限隔离
- 兼容性：支持 Chrome、Edge 浏览器
- 易用性：前端页面操作流程清晰，无复杂交互
- 可维护性：代码结构清晰，注释规范，便于团队协作

## 三、技术路线
### 1. 总体架构
采用 B/S 架构：浏览器 → 前端页面 → Spring Boot 后端 → MySQL 数据库

### 2. 主要技术
- 后端：Spring Boot、Spring Web、Spring Data JPA
- 前端：HTML5、CSS3、JavaScript、Vue
- 数据库：MySQL 8.0
- 版本控制：Git + GitHub 团队开发

### 3. 数据设计初步考虑
- 主要实体：用户、学生、教师、课程、成绩
- 关键数据表：
  - user（用户表）：用户ID、用户名称、密码、角色、创建时间
  - student（学生表）：学生ID、学生姓名、班级
  - teacher（教师表）：教师ID、教师姓名、学院
  - course（课程表）：课程ID、课程名称、教师ID
  - score（成绩表）：成绩记录ID、学生ID、课程ID、成绩、考试时间
