## 一、项目概述

### 1. 项目背景
传统学生成绩管理多采用 Excel 表格或人工纸质记录方式，普遍存在查询效率低、数据易出错、跨角色共享困难、统计分析繁琐等问题。为解决上述痛点，本项目拟开发一套轻量化的在线学生成绩管理系统，实现学生成绩的规范化、数字化管理，降低人工操作成本，提升成绩管理的整体效率。

### 2. 项目目标
1. 实现管理员账号的登录验证功能（基础鉴权）
2. 完成学生基础信息的增删改查（CRUD）核心操作
3. 实现成绩信息的增删改查（CRUD）核心操作
4. 支持按学号、课程名称、学生姓名单一条件查询成绩
5. 保障系统核心功能可稳定运行、可基础演示
6. 搭建规范的项目开发目录结构，完成 Git 仓库基础配置

### 3. 开发环境与条件
- 后端：Spring Boot 3.x、Spring Data JPA
- 前端：HTML5、CSS3、JavaScript、Bootstrap
- 数据库：MySQL 8.0
- 工具：IntelliJ IDEA、Git、GitHub、Postman
- 运行环境：JDK 17、Maven 3.8+

## 二、需求分析

### 1. 目标用户与使用场景
- 核心目标用户：系统管理员、课程教师
- 次要目标用户：辅导员
- 核心使用场景：
  - 教师：录入学生课程成绩、修改错误成绩、查询所教课程成绩
  - 管理员：维护学生基础信息、核验成绩数据、批量查询成绩
  - 辅导员：按班级 / 姓名查询学生整体成绩

### 2. 核心功能清单
1. 用户登录功能
2. 学生信息增删改查
3. 成绩信息增删改查
4. 多条件成绩查询功能

### 3. 非功能需求
- 性能：接口响应时间≤1秒
- 安全：仅登录用户可操作
- 兼容性：支持 Chrome、Edge 浏览器
- 前端页面操作流程清晰，无复杂交互

---

## 三、技术路线

### 1. 总体架构
采用 B/S 架构：浏览器 → 前端页面 → Spring Boot 后端 → MySQL 数据库

### 2. 主要技术
- 后端：Spring Boot、Spring Web、JPA、Lombok
- 前端：Bootstrap、原生 JS
- 数据库：MySQL 8.0
- 版本控制：Git + GitHub

### 3. 数据设计初步考虑
- 主要实体：用户、学生、成绩
- 数据表：user（用户）、student（学生）、score（成绩）
